# 선택 정렬 (Selection Sort)

## 개요

선택 정렬은 정렬되지 않은 구간에서 가장 작은(또는 가장 큰) 원소를 **선택**하여 정렬된 구간의 맨 끝과 교환하는 방식을 반복하는 알고리즘이다. 매 패스마다 "최솟값을 골라 제자리에 놓는다"는 점에서 이름이 붙었다.

---

## 동작 원리

1. 전체 배열에서 최솟값의 인덱스를 찾는다.
2. 해당 최솟값을 현재 패스의 시작 원소와 교환한다.
3. 정렬된 구간을 한 칸 확장하고, 나머지 미정렬 구간에 대해 위 과정을 반복한다.
4. 미정렬 구간의 크기가 1이 되면 정렬이 완료된다.

### 예시

```
초기 배열: [5, 3, 8, 1, 2]

--- 1번째 패스 (범위: 0~4) ---
최솟값: 1 (index 3) → index 0과 교환
[1, 3, 8, 5, 2]  ← 1 확정

--- 2번째 패스 (범위: 1~4) ---
최솟값: 2 (index 4) → index 1과 교환
[1, 2, 8, 5, 3]  ← 2 확정

--- 3번째 패스 (범위: 2~4) ---
최솟값: 3 (index 4) → index 2와 교환
[1, 2, 3, 5, 8]  ← 3 확정

--- 4번째 패스 (범위: 3~4) ---
최솟값: 5 (index 3) → 교환 불필요
[1, 2, 3, 5, 8]  ← 5 확정

정렬 완료: [1, 2, 3, 5, 8]
```

## 구현 (Java)

### 기본 구현

```java
public static void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        int temp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = temp;
    }
}
```

### 최적화 구현 (최솟값·최댓값 동시 선택)

한 패스에서 최솟값과 최댓값을 동시에 찾아 양쪽 끝에 배치하면 패스 횟수를 절반으로 줄일 수 있다.

```java
public static void selectionSortOptimized(int[] arr) {
    int left = 0, right = arr.length - 1;
    while (left < right) {
        int minIdx = left, maxIdx = left;
        for (int i = left + 1; i <= right; i++) {
            if (arr[i] < arr[minIdx]) minIdx = i;
            if (arr[i] > arr[maxIdx]) maxIdx = i;
        }
        // 최솟값을 왼쪽 끝으로
        int temp = arr[left];
        arr[left] = arr[minIdx];
        arr[minIdx] = temp;
        // maxIdx가 left를 가리키고 있었다면 swap으로 인해 위치가 바뀜
        if (maxIdx == left) maxIdx = minIdx;
        // 최댓값을 오른쪽 끝으로
        temp = arr[right];
        arr[right] = arr[maxIdx];
        arr[maxIdx] = temp;
        left++;
        right--;
    }
}
```

## 시간 복잡도 & 공간 복잡도

| 경우 | 시간 복잡도 | 설명 |
|------|------------|------|
| 최선 (Best) | O(n²) | 이미 정렬된 배열이어도 비교 횟수 동일 |
| 평균 (Average) | O(n²) | 무작위 배열 |
| 최악 (Worst) | O(n²) | 역순으로 정렬된 배열 |

- **공간 복잡도**: O(1) — 추가 메모리 없이 제자리(in-place) 정렬
- 비교 횟수는 항상 n(n-1)/2로 일정하지만, **교환 횟수는 최대 n-1번**으로 버블 정렬보다 적다.

---

## 안정성 (Stability)

선택 정렬은 **불안정 정렬(Unstable Sort)** 이다.

최솟값을 현재 위치의 원소와 교환할 때, 동일한 값을 가진 원소들의 상대적 순서가 뒤바뀔 수 있다.

```
예시: [3a, 3b, 1]  (3a, 3b는 값이 같지만 순서가 다른 원소)

1번째 패스: 최솟값 1 → index 0(3a)과 교환
결과: [1, 3b, 3a]  ← 3a와 3b의 순서가 뒤바뀜
```

## 다른 정렬 알고리즘과의 비교

| 알고리즘 | 평균 시간 | 최악 시간 | 공간 | 안정 |
|----------|----------|----------|------|------|
| 버블 정렬 | O(n²) | O(n²) | O(1) | ✅ |
| 선택 정렬 | O(n²) | O(n²) | O(1) | ❌ |
| 삽입 정렬 | O(n²) | O(n²) | O(1) | ✅ |
| 병합 정렬 | O(n log n) | O(n log n) | O(n) | ✅ |
| 퀵 정렬 | O(n log n) | O(n²) | O(log n) | ❌ |

## 특징 및 실전 활용

**장점**
- 구현이 단순하고 직관적이다.
- 제자리 정렬(in-place)로 추가 메모리가 필요 없다.
- **교환(swap) 횟수가 최대 n-1번**으로 매우 적다. 쓰기 비용이 비싼 환경(예: Flash 메모리)에서 유리하다.

**단점**
- 최선의 경우에도 O(n²)로, 조기 종료가 불가능하다.
- 불안정 정렬이므로 동일 키를 가진 원소의 순서가 보장되지 않는다.
- 같은 O(n²) 계열에서도 삽입 정렬보다 실제 성능이 떨어지는 경우가 많다.

**실전 활용**
- 실전 코딩 테스트에서 직접 사용하는 경우는 드물다.
- 다만 **교환 횟수 최소화**가 목적인 문제나, 정렬 과정 자체를 시뮬레이션하는 문제에서 등장할 수 있다.

---

## 관련 개념

- **제자리 정렬(In-place Sort)**: 입력 배열 외에 O(1)의 추가 공간만 사용하는 정렬.
- **불안정 정렬(Unstable Sort)**: 동일한 키를 가진 원소들의 원래 순서가 보장되지 않는 정렬.
- **교환 횟수**: 선택 정렬의 교환 횟수는 최대 n-1번으로, O(n²) 계열 중 가장 적다.
