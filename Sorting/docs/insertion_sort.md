# 삽입 정렬 (Insertion Sort)

## 개요

삽입 정렬은 정렬된 구간을 유지하면서 미정렬 구간의 원소를 하나씩 꺼내 올바른 위치에 **삽입**하는 알고리즘이다. 카드 게임에서 손패를 정렬하는 방식과 유사하다.

---

## 동작 원리

1. 두 번째 원소부터 시작하여 현재 원소를 `key`로 저장한다.
2. `key`보다 큰 원소들을 한 칸씩 오른쪽으로 이동시킨다.
3. `key`보다 작거나 같은 원소를 만나면 그 오른쪽에 `key`를 삽입한다.
4. 배열의 마지막 원소까지 반복한다.

### 예시

```
초기 배열: [5, 3, 8, 1, 2]

--- i=1, key=3 ---
5 > 3 → 5를 오른쪽으로 이동
[_, 5, 8, 1, 2] → index 0에 3 삽입
[3, 5, 8, 1, 2]

--- i=2, key=8 ---
5 < 8 → 이동 없음
[3, 5, 8, 1, 2]

--- i=3, key=1 ---
8 > 1, 5 > 1, 3 > 1 → 모두 오른쪽으로 이동
[_, 3, 5, 8, 2] → index 0에 1 삽입
[1, 3, 5, 8, 2]

--- i=4, key=2 ---
8 > 2, 5 > 2, 3 > 2 → 오른쪽으로 이동
1 < 2 → index 1에 2 삽입
[1, 2, 3, 5, 8]

정렬 완료: [1, 2, 3, 5, 8]
```

## 구현 (Java)

### 기본 구현

```java
public static void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

### 최적화 구현 (이진 탐색 삽입)

삽입 위치를 이진 탐색으로 찾아 비교 횟수를 O(log n)으로 줄인다. 단, 이동(shift) 횟수는 동일하므로 전체 시간 복잡도는 O(n²)이다.

```java
public static void binaryInsertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int lo = 0, hi = i;
        // key가 들어갈 위치를 이진 탐색
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (arr[mid] <= key) lo = mid + 1;
            else hi = mid;
        }
        // lo ~ i-1 구간을 한 칸씩 오른쪽으로 이동
        for (int j = i; j > lo; j--) {
            arr[j] = arr[j - 1];
        }
        arr[lo] = key;
    }
}
```

## 시간 복잡도 & 공간 복잡도

| 경우 | 시간 복잡도 | 설명 |
|------|------------|------|
| 최선 (Best) | O(n) | 이미 정렬된 배열 — 비교만 하고 이동 없음 |
| 평균 (Average) | O(n²) | 무작위 배열 |
| 최악 (Worst) | O(n²) | 역순으로 정렬된 배열 |

- **공간 복잡도**: O(1) — 제자리(in-place) 정렬
- 거의 정렬된 배열에서는 실제 성능이 O(n)에 가까워 **실용적으로 빠른 알고리즘**이다.

---

## 안정성 (Stability)

삽입 정렬은 **안정 정렬(Stable Sort)** 이다.

`arr[j] > key` 조건에서 등호를 포함하지 않으므로 같은 값의 원소는 이동시키지 않는다. 따라서 동일 값의 상대적 순서가 보존된다.

## 다른 정렬 알고리즘과의 비교

| 알고리즘 | 평균 시간 | 최악 시간 | 공간 | 안정 |
|----------|----------|----------|------|------|
| 버블 정렬 | O(n²) | O(n²) | O(1) | ✅ |
| 선택 정렬 | O(n²) | O(n²) | O(1) | ❌ |
| 삽입 정렬 | O(n²) | O(n²) | O(1) | ✅ |
| 병합 정렬 | O(n log n) | O(n log n) | O(n) | ✅ |
| 퀵 정렬 | O(n log n) | O(n²) | O(log n) | ❌ |

## 특징 및 실전 활용

**장점**
- 구현이 단순하고 직관적이다.
- 제자리 정렬(in-place), 안정 정렬이다.
- **거의 정렬된 배열에서 매우 빠르다** — 실제로 Tim Sort의 핵심 구성 요소.
- 소규모 배열(n ≤ 20 수준)에서는 퀵/병합보다 오버헤드가 적어 빠를 수 있다.

**단점**
- 평균·최악 O(n²)로 대규모 데이터에 부적합하다.
- 이동(shift) 연산이 많아 배열 크기가 커질수록 느려진다.

**실전 활용**
- Java의 `Arrays.sort(Object[])` 내부에서 사용하는 **Tim Sort**가 삽입 정렬 + 병합 정렬의 혼합이다.
- 거의 정렬된 데이터를 다루는 문제에서 직접 활용 가능하다.
- 온라인 알고리즘(Online Algorithm): 데이터가 하나씩 들어올 때도 정렬 상태를 유지할 수 있다.

## 관련 개념

- **역위(Inversion)**: 삽입 정렬의 총 이동 횟수 = 배열의 역위 수. 역위가 적을수록 빠르다.
- **Tim Sort**: 삽입 정렬(소규모 구간) + 병합 정렬(전체 병합)의 하이브리드. Java, Python 표준 정렬에 사용.
- **온라인 알고리즘**: 데이터를 순서대로 받으면서 정렬 상태를 유지할 수 있는 성질.
