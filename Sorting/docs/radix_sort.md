# 기수 정렬 (Radix Sort)

## 개요

기수 정렬은 원소를 자릿수(digit) 단위로 분류하여 정렬하는 알고리즘이다. 가장 낮은 자리(LSD)부터 또는 가장 높은 자리(MSD)부터 각 자릿수에 대해 안정 정렬(계수 정렬)을 반복 적용한다. 비교 연산을 사용하지 않아 O(n)에 근접한 성능을 낼 수 있다.

---

## 동작 원리 (LSD 방식)

1. 가장 낮은 자리(1의 자리)부터 시작한다.
2. 해당 자릿수를 기준으로 **안정 정렬(계수 정렬)** 을 수행한다.
3. 다음 자리(10의 자리, 100의 자리...)로 이동하여 반복한다.
4. 최대 자릿수까지 반복하면 정렬이 완료된다.

> **안정 정렬을 사용해야 하는 이유**: 낮은 자리에서 정렬된 순서를 높은 자리 정렬 시 유지해야 올바른 결과가 나오기 때문이다.

### 예시

```
초기 배열: [170, 45, 75, 90, 802, 24, 2, 66]

--- 1의 자리 기준 정렬 ---
0: [170, 90]
2: [802, 2]
4: [24]
5: [45, 75]
6: [66]
결과: [170, 90, 802, 2, 24, 45, 75, 66]

--- 10의 자리 기준 정렬 ---
0: [802, 2]       (10의 자리 = 0)
2: [24]
4: [45]
6: [66]
7: [170, 75]
9: [90]
결과: [802, 2, 24, 45, 66, 170, 75, 90]

--- 100의 자리 기준 정렬 ---
0: [2, 24, 45, 66, 75, 90]  (100의 자리 = 0)
1: [170]
8: [802]
결과: [2, 24, 45, 66, 75, 90, 170, 802]

정렬 완료: [2, 24, 45, 66, 75, 90, 170, 802]
```

## 구현 (Java)

### LSD 기수 정렬 (10진수)

```java
public static void radixSort(int[] arr) {
    int max = Arrays.stream(arr).max().getAsInt();
    // 자릿수마다 계수 정렬 수행
    for (int exp = 1; max / exp > 0; exp *= 10) {
        countingSortByDigit(arr, exp);
    }
}

private static void countingSortByDigit(int[] arr, int exp) {
    int n = arr.length;
    int[] output = new int[n];
    int[] count = new int[10];  // 0~9 자릿수

    // 현재 자릿수 카운트
    for (int val : arr) count[(val / exp) % 10]++;

    // 누적합
    for (int i = 1; i < 10; i++) count[i] += count[i - 1];

    // 역순 배치 (안정성 유지)
    for (int i = n - 1; i >= 0; i--) {
        int digit = (arr[i] / exp) % 10;
        output[--count[digit]] = arr[i];
    }

    System.arraycopy(output, 0, arr, 0, n);
}
```

### 비트 기반 기수 정렬 (2진수, 고성능)

기수(base)를 256(2⁸)으로 잡으면 패스 횟수가 4회로 줄어 실전에서 매우 빠르다.

```java
public static void radixSort256(int[] arr) {
    int n = arr.length;
    int[] output = new int[n];
    for (int shift = 0; shift < 32; shift += 8) {
        int[] count = new int[256];
        for (int val : arr) count[(val >> shift) & 0xFF]++;
        for (int i = 1; i < 256; i++) count[i] += count[i - 1];
        for (int i = n - 1; i >= 0; i--) {
            output[--count[(arr[i] >> shift) & 0xFF]] = arr[i];
        }
        System.arraycopy(output, 0, arr, 0, n);
    }
}
```

## 시간 복잡도 & 공간 복잡도

| 경우 | 시간 복잡도 | 설명 |
|------|------------|------|
| 최선 (Best) | O(d(n + k)) | d = 자릿수, k = 기수(진법) |
| 평균 (Average) | O(d(n + k)) | 항상 동일 |
| 최악 (Worst) | O(d(n + k)) | 항상 동일 |

- **공간 복잡도**: O(n + k)
- 정수의 최대 자릿수 d = O(log_k(max_value))
- 10진수 기준: d ≤ 10 (10억 이하), k = 10 → 사실상 **O(n)**
- k = 256일 때: d = 4 (32비트 정수), 더욱 빠름

## 안정성 (Stability)

기수 정렬은 **안정 정렬(Stable Sort)** 이다.

각 자릿수 정렬에 안정 정렬(계수 정렬)을 사용하므로 동일 값의 상대적 순서가 보존된다. 이 안정성이 낮은 자리의 정렬 결과를 보존하여 최종 정렬이 올바르게 됨을 보장한다.

## 다른 정렬 알고리즘과의 비교

| 알고리즘 | 평균 시간 | 최악 시간 | 공간 | 안정 | 비고 |
|----------|----------|----------|------|------|------|
| 병합 정렬 | O(n log n) | O(n log n) | O(n) | ✅ | 비교 기반 |
| 퀵 정렬 | O(n log n) | O(n²) | O(log n) | ❌ | 비교 기반 |
| 계수 정렬 | O(n + k) | O(n + k) | O(k) | ✅ | 범위 제한 필요 |
| 기수 정렬 | O(d(n + k)) | O(d(n + k)) | O(n + k) | ✅ | 자릿수 기반 |

## 특징 및 실전 활용

**장점**
- 조건이 맞을 때 O(n)에 가까운 성능을 낸다.
- 값의 범위가 매우 크더라도 자릿수(d)가 작으면 효율적이다. (계수 정렬의 한계 극복)
- 안정 정렬이다.

**단점**
- 정수 또는 고정 길이 문자열 등 자릿수 개념이 있는 데이터에만 적용 가능하다.
- 구현이 계수 정렬보다 복잡하다.
- 음수 처리가 추가로 필요하다 (부호 비트 별도 처리 또는 오프셋 적용).
- 실전 코딩 테스트에서 직접 구현하는 경우는 드물지만, 원리 이해가 중요하다.

**실전 활용**
- 값의 범위는 크지만 자릿수가 고정된 데이터 (전화번호, 날짜, IP 주소 등) 정렬.
- 문자열 정렬에서 LSD/MSD 기수 정렬 응용.
- 대용량 데이터 정렬 시스템에서 퀵/병합 정렬의 대안.

## LSD vs MSD

| 방식 | 시작 자리 | 특징 |
|------|----------|------|
| **LSD** (Least Significant Digit) | 가장 낮은 자리 | 구현 단순, 전체 정렬에 적합 |
| **MSD** (Most Significant Digit) | 가장 높은 자리 | 재귀적, 부분 정렬·조기 종료 가능, 문자열 정렬에 자연스러움 |

---

## 관련 개념

- **기수(Radix)**: 수를 표현하는 진법의 밑수. 10진수라면 k=10, 2진수라면 k=2, 256진수라면 k=256.
- **LSD 기수 정렬**: 낮은 자리부터 정렬. 반복 구현이 단순하고 안정성이 자연스럽게 유지됨.
- **MSD 기수 정렬**: 높은 자리부터 재귀적으로 정렬. 사전순 문자열 정렬에 적합.
- **계수 정렬과의 관계**: 기수 정렬은 각 자릿수에 계수 정렬을 적용하는 방식으로, 계수 정렬의 확장이라 볼 수 있다.
