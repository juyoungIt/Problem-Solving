# 버블 정렬 (Bubble Sort)

## 개요

버블 정렬은 인접한 두 원소를 비교하여 순서가 잘못되어 있으면 교환하는 방식을 반복해 정렬하는 알고리즘이다. 각 패스(pass)마다 가장 큰 원소가 배열의 끝으로 "거품처럼 올라온다"는 점에서 이름이 붙었다.

---

## 동작 원리

1. 배열의 첫 번째 원소부터 시작하여 인접한 두 원소를 비교한다.
2. 앞 원소가 뒤 원소보다 크면 두 원소를 교환(swap)한다.
3. 한 패스가 끝나면 배열의 마지막에 가장 큰 원소가 위치한다.
4. 이미 정렬된 마지막 원소를 제외하고 위 과정을 반복한다.
5. 더 이상 교환이 발생하지 않으면 정렬이 완료된 것이다.

### 예시

```
초기 배열: [5, 3, 8, 1, 2]

--- 1번째 패스 ---
[5, 3, 8, 1, 2] → 5 > 3 → 교환 → [3, 5, 8, 1, 2]
[3, 5, 8, 1, 2] → 5 < 8 → 유지 → [3, 5, 8, 1, 2]
[3, 5, 8, 1, 2] → 8 > 1 → 교환 → [3, 5, 1, 8, 2]
[3, 5, 1, 8, 2] → 8 > 2 → 교환 → [3, 5, 1, 2, 8]  ← 8 확정

--- 2번째 패스 ---
[3, 5, 1, 2, 8] → 3 < 5 → 유지
[3, 5, 1, 2, 8] → 5 > 1 → 교환 → [3, 1, 5, 2, 8]
[3, 1, 5, 2, 8] → 5 > 2 → 교환 → [3, 1, 2, 5, 8]  ← 5 확정

--- 3번째 패스 ---
[3, 1, 2, 5, 8] → 3 > 1 → 교환 → [1, 3, 2, 5, 8]
[1, 3, 2, 5, 8] → 3 > 2 → 교환 → [1, 2, 3, 5, 8]  ← 3 확정

--- 4번째 패스 ---
[1, 2, 3, 5, 8] → 1 < 2 → 유지                     ← 2 확정

정렬 완료: [1, 2, 3, 5, 8]
```

## 구현 (Java)

### 기본 구현

```java
public static void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```

### 최적화 구현 (조기 종료)

한 패스에서 교환이 한 번도 발생하지 않으면 이미 정렬된 상태이므로 조기에 종료한다.

```java
public static void bubbleSortOptimized(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        if (!swapped) break;  // 교환 없음 → 정렬 완료
    }
}
```

## 시간 복잡도 & 공간 복잡도

| 경우 | 시간 복잡도 | 설명 |
|------|------------|------|
| 최선 (Best) | O(n) | 이미 정렬된 배열 (조기 종료 최적화 적용 시) |
| 평균 (Average) | O(n²) | 무작위 배열 |
| 최악 (Worst) | O(n²) | 역순으로 정렬된 배열 |

- **공간 복잡도**: O(1) — 추가 메모리 없이 제자리(in-place) 정렬

---

## 안정성 (Stability)

버블 정렬은 **안정 정렬(Stable Sort)** 이다.

동일한 값을 가진 원소들의 상대적 순서가 정렬 후에도 유지된다. 인접 비교 시 `arr[j] > arr[j+1]`처럼 **등호를 포함하지 않는** 조건을 쓰기 때문에 같은 값이면 교환하지 않아 원래 순서가 보존된다.

## 다른 정렬 알고리즘과의 비교

| 알고리즘 | 평균 시간 | 최악 시간 | 공간 | 안정 |
|----------|----------|----------|------|------|
| 버블 정렬 | O(n²) | O(n²) | O(1) | ✅ |
| 선택 정렬 | O(n²) | O(n²) | O(1) | ❌ |
| 삽입 정렬 | O(n²) | O(n²) | O(1) | ✅ |
| 병합 정렬 | O(n log n) | O(n log n) | O(n) | ✅ |
| 퀵 정렬 | O(n log n) | O(n²) | O(log n) | ❌ |

## 특징 및 실전 활용

**장점**
- 구현이 단순하고 직관적이다.
- 제자리 정렬(in-place)로 추가 메모리가 필요 없다.
- 안정 정렬이다.
- 조기 종료 최적화 시 거의 정렬된 배열에서는 O(n)에 가깝다.

**단점**
- 평균·최악 시간 복잡도가 O(n²)로 대규모 데이터에 부적합하다.
- 교환(swap) 연산이 선택 정렬에 비해 훨씬 많이 발생한다.

**실전 활용**
- 실전 코딩 테스트에서는 직접 사용하는 경우가 드물다.
- 다만 버블 정렬의 **패스 횟수** 또는 **교환 횟수**를 이용하는 문제 유형이 존재한다.
  - 예: BOJ 1377 — 버블 정렬이 몇 번의 패스 만에 끝나는지 구하기

## 관련 개념

- **역위(Inversion)**: 배열에서 `i < j`이지만 `arr[i] > arr[j]`인 쌍. 버블 정렬의 교환 횟수 = 전체 역위의 수.
- **조기 종료(Early Termination)**: 한 패스에서 교환이 없으면 이미 정렬된 상태.
